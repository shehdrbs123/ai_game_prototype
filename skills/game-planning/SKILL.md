# Game Planning

## Overview

Turn vague game ideas into execution-ready planning artifacts.
Drive decisions with explicit assumptions, tradeoffs, constraints, and measurable success criteria.

## Workflow

1. Clarify objective and constraints.
2. Define the player, fantasy, and value proposition.
3. Design the core loop and progression loop.
4. Select scope for MVP and post-MVP.
5. Build milestone and staffing plan.
6. Evaluate risks and mitigation.
7. Deliver concise artifacts in the requested format.

## Output Rules

Use concise Korean by default; keep technical terms in English when clearer.
State conclusion first, then evidence.
Avoid tables unless the user explicitly asks for them.
For each major decision, include:
- Assumption
- Option considered
- Reason for choice
- Validation metric

## Core Deliverables

Pick only what the user needs:
- One-pager concept brief
- Core loop spec
- MVP scope list (Must/Should/Could)
- Content and progression outline
- Economy and reward structure
- Milestone plan (Alpha/Beta/Launch)
- Risk register with mitigation
- Playtest plan and KPI set

## Decision Heuristics

When information is missing, do not guess. Ask for the smallest missing input.
Prefer one strong differentiator over broad feature sprawl.
For early prototypes, prioritize retention signal over content volume.
If schedule or budget is constrained, cut features before lowering core loop quality.
Tie each feature to a target player motivation (mastery, expression, collection, social, narrative).

## Artifact Templates

Use templates in [references/templates.md](references/templates.md) when generating:
- Concept one-pager

- Playtest KPI sheet

## Quality Gate

Before finalizing outputs, check:
- Is target audience explicit?
- Is core loop testable in under 10 minutes?
- Is MVP scope feasible for stated team and timeline?
- Are launch risks paired with concrete mitigation?
- Are KPIs measurable from available telemetry?
