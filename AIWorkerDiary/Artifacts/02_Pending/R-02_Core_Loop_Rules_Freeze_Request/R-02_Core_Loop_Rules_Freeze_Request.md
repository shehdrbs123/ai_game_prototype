# R-02 코어 루프 규칙 고정 요청서 (개발 착수용)

- Version: v0.2
- Date: 2026-03-07
- Owner: Planning
- Target: MVP 0

## 1. 목적

코어 루프를 6단계로 고정하고, 위험도(0~3) 기반 사망 손실/회수/보험 규칙을 구현 가능한 수준으로 명세한다.

## 2. 범위

- 포함
- 코어 루프 6단계 고정
- 던전 입장 제물 시스템(MVP 0)
- 위험도(0~3)별 사망 손실/회수 규칙
- 위험도 3 전용 보험 규칙
- 제외
- 메타 비용 밸런싱/상한 설계(후속 요청서)
- 멀티플레이 연동
- 시즌/이벤트 강제 던전 오버라이드

## 3. 코어 루프 (고정)

1. Town 준비(장비/제작/업그레이드/건설)
2. 제단 세팅(제물 3칸 + 옥 1칸) 후 던전 진입
3. 전투 및 아이템 수집
4. 탈출 시도
5. 탈출 성공 시 보상 확보
6. 재투자(시설 건축 포함) 후 다음 run

## 4. 제물 시스템 (MVP 0 고정)

1. 제물 슬롯은 정확히 3칸, 옥 슬롯은 1칸
2. 옥은 0개 또는 1개만 허용
3. 출정 시 제물/옥은 전량 소모
4. 위험도 결정
- 옥 없음: 위험도 0
- 옥 등급 1: 위험도 1
- 옥 등급 2: 위험도 2
- 옥 등급 3: 위험도 3
5. 던전 결정
- 제물 3개의 `targetDungeon` 다수결
- 동률 시 seed 기반 랜덤 1회

참조 문서: `AIWorkerDiary/Artifacts/02_Pending/Dungeon_Entrance_System/simplePlan.md`

## 5. 사망 손실/회수 규칙 (고정)

1. 위험도 0
- 다음 던전 입장 시 플레이어 스폰 근처 시체 1개 생성
- 사망 시점 소지품 100% 회수 가능
2. 위험도 1
- 사망 위치 시체 1개 생성
- 소지품 100% 회수 가능
3. 위험도 2
- 사망 위치 시체 1개 생성
- 소지품 50% 회수 가능
- 회수 대상 선정은 seed 기반 랜덤(재현 가능)
4. 위험도 3
- 시체 회수 없음(0%)
- 보험 가입 시에만 복구 로직 적용

## 6. 보험 규칙 (위험도 3 전용)

1. 가입 시점: 던전 입장 직전
2. 보험료 기준: 들고간 아이템 기준으로 산정
3. 사망 시 복구율: 1/3
4. 복구 아이템 선정: seed 기반 랜덤
5. 보험 미가입 시 복구 0%

## 7. 구현 요구사항

1. 데이터 모델
- `riskLevel`, `insuranceEnabled`, `insurancePremium` 저장
- run 시작 시 `runLoadoutSnapshot` 저장(보험/복구 기준 원본)
- 사망 시 `deathDropPool` 또는 동등 데이터 저장
2. 엔진 로직
- 사망 시 위험도 분기 처리
- 시체 생성 위치 정책 분기(스폰 근처/사망 위치/미생성)
- 회수/복구 대상 계산 로직 구현
3. UI
- 제단 화면: 위험도 프리뷰 표시
- 위험도 3: 보험 토글 + 예상 보험료 표시
- 결과 화면: 손실/회수/복구 내역 표시

## 8. 수용 기준 (Acceptance Criteria)

1. 위험도 0 사망 후 다음 run 시작 10초 내 시체 확인 가능
2. 위험도 1 사망 좌표 반경 허용 오차 내 시체 생성
3. 위험도 2에서 소지품 N개일 때 회수 가능 개수 `floor(N * 0.5)`
4. 위험도 3 미가입 시 회수/복구 아이템 0개
5. 위험도 3 가입 시 복구 개수 `floor(N / 3)`
6. 동일 seed + 동일 loadout 입력 시 회수/복구 결과 동일
7. 제물/옥 소모는 출정 시점 1회만 차감

## 9. 개발 작업 단위 (권장 순서)

1. 데이터 스키마 확장
2. 제단 입력/위험도 산출 로직 구현
3. 사망 처리 분기 + 시체 생성 정책 구현
4. 보험료 계산 + 복구 로직 구현
5. 제단/결과 UI 반영
6. 수용 기준 7개 시나리오 검증
