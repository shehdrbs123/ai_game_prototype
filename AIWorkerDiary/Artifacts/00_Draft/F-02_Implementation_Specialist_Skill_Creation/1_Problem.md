# Phase 1: Problem - Implementation Specialist Skill Creation

## 1. Problem Statement (문제 정의)
- **현상:** 현재 `manager-architect`, `game-data-expert` 등 설계 및 데이터 중심 스킬은 존재하나, 실제 복잡한 비즈니스 로직을 구현하고 기술 표준(XML 주석, SOLID, Manager Pattern)을 일관되게 적용하는 '구현 전문가' 페르소나가 부재함.
- **영향:** 코드 구현 시 매번 전역 지침을 다시 확인해야 하며, 구현 품질의 일관성이 떨어질 수 있음.

## 2. Requirements (요구사항)
- **Role Name:** `implementation-specialist`
- **Core Competencies:** 
    - ES6+ JavaScript 전문성.
    - Manager-based Architecture (DIContainer)에 대한 깊은 이해.
    - XML 스타일 주석 및 클린 코드 준수.
    - 리팩토링 및 버그 수정 최적화.
- **Trigger Condition:** "코드 구현해줘", "로직 작성해줘", "리팩토링해줘" 등 실제 파일 수정 및 생성이 수반되는 작업 시.

## 3. Goals (목표)
- `implementation-specialist` 스킬의 `SKILL.md` 작성 및 설치.
- 프로젝트의 `src/` 내 코드 작성 규칙을 내재화한 참조 문서(`references/`) 포함.

## 4. Constraints (제약 사항)
- `.gemini/skills/` 디렉토리 내에 위치해야 함.
- `skill-creator`의 `Concise is Key` 원칙을 준수하여 불필요한 토큰 낭비를 방지함.
